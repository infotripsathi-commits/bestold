🎉 Social Media Feature Added!

Store owners can now add YouTube, Facebook & Instagram links

⚠️ Not Seeing the New Features?

If you don't see the social media input fields on the Store Management page, your browser is showing you an old cached version. Follow the steps below to fix this.

1 Hard Refresh Your Browser

This forces your browser to download the latest version of the website:

  • Windows/Linux: Press Ctrl + Shift + R
  • Mac: Press Cmd + Shift + R
  • Alternative: Press Ctrl + F5 (Windows/Linux)

2 Clear Browser Cache

If hard refresh doesn't work, clear your browser cache:

  1. Press Ctrl + Shift + Delete (or Cmd + Shift + Delete on Mac)
  2. Select "Cached images and files"
  3. Choose time range: "Last hour" or "Last 24 hours"
  4. Click "Clear data"
  5. Restart your browser

3 Verify the Update Loaded

After refreshing, open the browser console to verify:

  1. Press F12 to open Developer Tools
  2. Click on the "Console" tab
  3. Navigate to the Store Management page
  4. Look for this message:
🔄 StoreManagementPage loaded - Version with Social Media Links v1.0
✅ Social Media section should be visible after Phone Number field

If you see this message, the page has loaded correctly!

✅ What You Should See

On Store Management Page (Seller):

  • Phone Number field
  • Social Media Links section (NEW!)
  • YouTube Channel input field
  • Facebook Page input field
  • Instagram Profile input field
  • Trade License Upload section

On Store Detail Page (Customer):

  • Store description and images
  • Connect With Us section (if store added links)
  • YouTube, Facebook, Instagram buttons

4 Still Not Working?

Try these additional steps:

  • Try a different browser (Chrome, Firefox, Safari, Edge)
  • Try incognito/private browsing mode
  • Check if you're logged in as a Seller account (not Buyer)
  • Verify you're on the correct page: /seller/store-management

Check Database (Advanced):

-- Run this in Supabase SQL Editor
SELECT column_name FROM information_schema.columns
WHERE table_name = 'stores'
AND column_name IN ('youtube_url', 'facebook_url', 'instagram_url');

Should return 3 rows. If not, the database migration needs to be applied.
